Verun Protocol Documentation

Verun is an Agent Trust Score Layer for autonomous agentic finance. Every trust evaluation is anchored on Starknet — producing a tamper-evident audit trail of agent behaviour across platforms.

⚠ BETA NOTICE
Early-stage MVP on Starknet Sepolia testnet. Anything shown here — contracts, APIs, validators, design, terminology — can change as the protocol evolves. Testnet-only: addresses and transaction hashes hold no economic value.

The problem

AI agents are beginning to transact autonomously with real money — investing, transferring, minting, ordering. But there is no shared trust infrastructure. How do humans verify that a given agent is trustworthy? How do platforms know which agents to accept? How do we audit autonomous decisions after the fact?

The solution

Verun provides three things:

  1. A trust score — bidirectional reputation for agents and platforms
  2. A validator consensus — 2-of-3 approval model for material actions
  3. On-chain anchoring — every approved evaluation written to Starknet for a tamper-evident audit trail

Why Starknet

  • Low fees per transaction — suitable for high-volume agent activity
  • Cairo's provable computation — fits trust-verification use cases
  • Maturing ecosystem and tooling

Architecture

Verun has three layers: a client-facing HTTP API, a validator consensus engine, and on-chain anchoring in Cairo smart contracts.

┌──────────┐ POST /api/evaluate ┌─────────────────────┐ │ Client │ ──────────────────────► │ Vercel Serverless │ └──────────┘ │ evaluate.js │ │ │ │ 1. Score vs gate │ │ 2. Validator vote │ │ 3. If approved → │ │ signed tx to │ │ TrustScore │ └──────────┬───────────┘ │ starknet.js v8 ▼ ┌─────────────────────┐ │ Starknet Sepolia │ │ │ │ TrustScore.cairo │ │ └ record_score() │ │ │ │ ValidatorRegistry │ └─────────────────────┘

Request flow

  1. Client submits an evaluation request with agentId, score, and operation
  2. API checks the score against the operation's gate threshold (see below)
  3. A 2-of-3 validator vote is simulated for consensus
  4. If the score clears the gate AND consensus is reached, the API signs a record_score transaction on the TrustScore contract
  5. The response returns the validator votes, consensus result, verdict, and (if approved) the real Starknet transaction hash

POST /api/evaluate

Full evaluation — if approved, submits a real record_score transaction on the TrustScore contract and returns the transaction hash.

Request

{
  "agentId": "agent-001",
  "score": 650,
  "operation": "transfer"
}

Response

{
  "agentId": "agent-001",
  "score": 650,
  "operation": "transfer",
  "requiredScore": 500,
  "meetsThreshold": true,
  "validators": [ ... ],
  "consensusReached": true,
  "approved": true,
  "timestamp": "...",
  "starknet": {
    "network": "Starknet Sepolia",
    "mode": "live",
    "contractAddress": "0x4d1f47e42dab...",
    "transactionHash": "0x2a155037b2743ca...",
    "explorer": "https://sepolia.voyager.online/tx/0x..."
  }
}

Notes

  • agentId must be ≤ 31 ASCII characters (felt252 limit). Longer IDs should be Poseidon-hashed off-chain before submission.
  • Score must be an integer in the range 0–1000.
  • The API submits the tx and returns the hash without waiting for confirmation. Clients can poll the returned explorer URL.
  • If chain-side env vars are missing or the chain call fails, the response includes "mode": "mock" and an anchorError instead of a real tx.

Validator Network

Verun uses a 2-of-3 validator consensus model. A request is approved only when at least two validators approve and the score clears the operation's gate threshold.

FAQ

Is this on mainnet?

No. Verun is currently on Starknet Sepolia testnet. Mainnet deployment is on the roadmap and will follow an independent security audit.

What's actually anchored on-chain?

For every approved evaluation, the API submits a record_score transaction to the TrustScore contract on Starknet Sepolia. The transaction records the agent_id, score, and timestamp — producing an immutable, timestamped record visible on Voyager.

How do I become a validator?

The Open Slot is available for institutional validators — European banks, custodians, and regulated compliance partners. Apply via bcpp.io/contact-us.

What does the trust score represent?

In the current MVP, scores shown in examples are illustrative. Future versions will calculate the score from on-chain history, validator attestations, and external compliance signals. The scoring formula is under design.

Can I integrate Verun into my agent app?

Yes — the API is public and documented above. A dedicated developer SDK is on the roadmap.

Is Verun MiCA-compliant?

Verun's architecture is designed to map to MiCA, FATF, and ESMA principles — but formal regulatory compliance requires external audits and KYC/AML integrations, which are on our roadmap. The current MVP is a technical building block, not a certified compliance product.
