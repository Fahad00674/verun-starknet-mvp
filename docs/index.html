Verun Protocol Documentation

Verun is an Agent Trust Score Layer for autonomous agentic finance. Every trust evaluation is anchored on Starknet, producing a tamper-evident audit trail of agent behaviour across platforms.

⚠ BETA NOTICE
Early-stage MVP on Starknet Sepolia testnet. Anything shown here — contracts, APIs, validators, design, terminology — can change as the protocol evolves. Testnet-only: addresses and transaction hashes hold no economic value.

The problem

AI agents are beginning to transact autonomously with real money — investing, transferring, minting, ordering. But there is no shared trust infrastructure. How do humans verify a given agent is trustworthy? How do platforms decide which agents to accept? How do we audit autonomous decisions after the fact?

The solution

Verun provides three things:

  1. A trust score — bidirectional reputation for agents and platforms.
  2. Validator consensus — 2-of-3 approval model for material actions.
  3. On-chain anchoring — every approved evaluation written to Starknet, producing a tamper-evident audit trail.

Why Starknet

  • Low fees per transaction — suitable for high-volume agent activity.
  • Cairo's provable computation — fits trust-verification use cases.
  • Maturing ecosystem and tooling — Voyager, ArgentX, Starknet.js.

Architecture

Verun acts as trust middleware between AI agents and on-chain execution. Trust scores require 2-of-3 validator consensus before any score update is anchored on-chain.

Request flow

  1. Client submits an evaluation request with agentId, score, and operation.
  2. The API checks the score against the operation's gate threshold.
  3. A 2-of-3 validator vote is tallied for consensus.
  4. If the score clears the gate and consensus is reached, the API signs a record_score transaction on the TrustScore contract.
  5. The response returns the validator votes, consensus result, verdict, and (if approved) the real Starknet transaction hash.

Notes

  • agentId must be ≤ 31 ASCII characters (felt252 limit).
  • score must be an integer in the range 0–1000.
  • The API submits the tx and returns the hash without waiting for confirmation.
  • If chain env vars are missing, the response includes "mode": "mock" and an anchorError.
  • Backend auto-retries NonceTooOld errors up to four times with jittered backoff.
⚠ AUTHENTICATION GAP
All API endpoints in the current MVP are unauthenticated. Any caller can submit evaluations or anchor scores for any agentId. A request-authentication boundary (scoped API key, agent-signed attestation, or Verun-issued JWT) is a deliverable in the "Security & ops baseline" workstream of the grant roadmap. Do not rely on the current endpoints for any production or user-asset context.

tokenforge Integration

Verun sits above tokenforge's TokenSuite as the trust gate for agent-initiated actions. tokenforge provides the white-label STO execution layer; Verun provides the trust attestation anchored on Starknet.

Division of responsibility

  • Verun — evaluates agent trust score, runs validator consensus, anchors the approved evaluation on Starknet.
  • tokenforge TokenSuite — executes the operation (mint, transfer, order) on its regulated Chain API, using Cashlink for crypto-securities registry and custody.

Integration pattern

  1. Agent wants to call a tokenforge Chain API endpoint (e.g. POST /transfer).
  2. Agent first calls Verun's POST /api/evaluate with its agentId, current score, and intended operation.
  3. Verun checks the operation gate and runs validator consensus.
  4. On approval, Verun anchors the attestation on Starknet and returns the transaction hash.
  5. Agent forwards the Verun attestation (tx hash + response) to tokenforge's Chain API alongside its request.
  6. tokenforge verifies the Verun attestation and executes the requested operation.

FAQ

Is this on mainnet?

No. Verun is currently on Starknet Sepolia testnet. Mainnet deployment is on the roadmap and will follow an independent security audit.

What's actually anchored on-chain?

For every approved evaluation, the API submits a record_score transaction to the TrustScore contract on Starknet Sepolia. The transaction records agent_id, score, and timestamp — producing an immutable, timestamped record visible on Voyager.

How do I become a validator?

The Open Slot is available for institutional validators — European banks, custodians, and regulated compliance partners. Apply via bcpp.io/contact-us.

What does the trust score represent?

In the current MVP, scores shown in examples are illustrative. Future versions will derive the score from on-chain history, validator attestations, and external compliance signals. The scoring formula is under design.

Can I integrate Verun into my agent app?

Yes — the API is public and documented above. A dedicated developer SDK is on the roadmap.

Is Verun MiCA-compliant?

Verun's architecture is designed to map to MiCA, FATF, and ESMA principles — but formal regulatory compliance requires external audits and KYC/AML integrations, which are on our roadmap. The current MVP is a technical building block, not a certified compliance product.
